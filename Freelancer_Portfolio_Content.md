> Conducted a complete manual QA cycle for ShopEase, a B2C eCommerce web platform, covering Release v2.3.0. The engagement spanned 15 business days and produced a full suite of professional QA documentation from test planning through to executive test summary reporting.
>
> **My Role**
>
> Acting as the sole QA Engineer, I was responsible for the entire QA process: requirements analysis, test strategy, test case design, execution, defect management, and test closure reporting.
>
> **Key Activities**
>
> *Test Planning:* Reviewed business requirements and functional specifications to define a risk-based test strategy. Produced a formal Test Plan covering in/out scope, test types, entry/exit criteria, browser/device matrix, test data requirements, risk register, and test schedule.
>
> *Test Case Design:* Authored 20 detailed test cases using Boundary Value Analysis (BVA) and Equivalence Partitioning (EP). Modules covered include: User Authentication, Homepage & Navigation, Product Search & Filtering, Shopping Cart, Checkout Flow, Payment Integration, Responsive Layout, and Form Validation.
>
> *Test Execution:* Executed all 20 test cases across Chrome, Firefox, Safari, and mobile viewports. Achieved an 85% pass rate (17 pass / 2 fail / 1 blocked). All failures linked to specific JIRA defect tickets with full reproduction evidence.
>
> *Defect Reporting:* Raised 3 professional bug reports, including a Critical-severity payment decline error handling defect (abandoned checkout risk), a Major cart total miscalculation (financial discrepancy risk), and a Major mobile responsive overflow (mobile UX/SEO risk). Each report includes business impact analysis and developer recommendations.
>
> *Test Summary:* Delivered a comprehensive Test Summary Report with go/no-go release recommendation, risk analysis, and QA sign-off process — used by the Product Owner and development lead to make the release decision.
>
> **Results**
>
> - 3 pre-production defects identified, preventing a potentially faulty release
> - Critical payment flow bug discovered before go-live, avoiding estimated £4,800+/day revenue impact
> - Cart total miscalculation documented with root cause — developer resolved in <24 hours
> - All QA artefacts delivered within the agreed 15-day timeline
>
