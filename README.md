## 🔬 Testing Types Covered

- ✅ **Functional Testing** — Verifying each feature operates per business requirements
- ✅ **Regression Testing** — Ensuring new changes do not break existing functionality
- ✅ **UI / UX Testing** — Validating visual elements, layout consistency, and accessibility
- ✅ **Boundary Value Analysis (BVA)** — Testing edge cases at input boundaries
- ✅ **Equivalence Partitioning (EP)** — Grouping valid/invalid input classes
- ✅ **Negative Testing** — Validating system behaviour with invalid/unexpected inputs
- ✅ **Responsive / Cross-Device Testing** — Desktop, tablet, and mobile viewports
- ✅ **Exploratory Testing** — Ad-hoc sessions beyond scripted test cases
- ✅ **End-to-End (E2E) Testing** — Full user journey from browse to purchase

## 🧭 Testing Approach

### Phase 1 — Test Planning
Review business requirements, functional specifications, and UI/UX wireframes. Define scope, test strategy, entry/exit criteria, and risk register. Produced: **Test Plan Document**.

### Phase 2 — Test Design
Derive test scenarios from user stories using BVA and EP techniques. Write detailed, reusable test cases with clear preconditions, steps, and expected results. Produced: **Test Cases Spreadsheet**.

### Phase 3 — Test Execution
Execute all test cases per the defined test schedule. Log pass/fail/blocked statuses in TestRail. Raise defect tickets in JIRA with full reproduction steps and evidence.

### Phase 4 — Defect Management
Triage bugs with development team. Re-test resolved defects. Maintain defect lifecycle (New → Open → In Progress → Fixed → Re-test → Closed).

### Phase 5 — Test Closure
Compile test execution metrics, defect summary, risk analysis, and release recommendation. Produced: **Test Summary Report** and **QA Sign-Off**.
